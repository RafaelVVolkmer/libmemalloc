# Code of Conduct for libmemalloc ü§ù

This document describes the expected behavior of everyone interacting with the
**libmemalloc** project, including maintainers, contributors, and users, in all
project spaces (GitHub issues, pull requests, discussions, and any other
communication channels related to the project).

The goal is to keep the project a welcoming, respectful, and technically
focused environment.

---

## 1. Scope üåç

This Code of Conduct applies to:

- The official libmemalloc GitHub repository  
  (`https://github.com/RafaelVVolkmer/libmemalloc`), including:
  - Issues
  - Pull Requests
  - Discussions
  - Wiki / documentation
- Any other communication where the project is being coordinated or discussed
  and where participants identify as part of the libmemalloc community.

---

## 2. Expected Behavior ‚úÖ

Participants are expected to:

- Be **respectful** and **professional** in all interactions.
- Focus on **technical arguments**, not on individuals.
- Assume good intentions and try to **collaborate** to reach the best solution.
- Give and receive feedback in a **constructive** way.
- Respect different experience levels, backgrounds, and perspectives.
- Keep communication **clear**, **relevant**, and **on-topic**.

Examples of positive behavior include:

- Helping new contributors understand the codebase or the contribution process.
- Clearly explaining trade-offs and design decisions.
- Providing minimal reproducer cases and precise technical information when
  reporting bugs.

---

## 3. Unacceptable Behavior üö´

The following behavior is not acceptable in the libmemalloc community:

- Personal attacks, insults, or demeaning comments.
- Harassment of any kind (verbal, written, or implicit).
- Discriminatory language or behavior related to gender, sexual orientation,
  disability, physical appearance, body size, race, ethnicity, religion,
  nationality, or any other personal characteristic.
- Threats, intimidation, or encouraging self-harm or harm to others.
- Persistent disruption of discussions or ignoring requests from maintainers
  to change behavior.
- Sharing private information of others without explicit consent.

---

## 4. Reporting Issues üì¨

If you experience or witness behavior that violates this Code of Conduct, you
can report it privately to the maintainer:

- **Email:** `rafael.v.volkmer@gmail.com`

When reporting, please include (when possible):

- A short description of what happened.
- Links (issues, PRs, comments, discussion threads) or screenshots.
- The approximate date and time of the incident.
- Any additional context that may help understand the situation.

All reports will be treated with **confidentiality**. The maintainer will
review the case and decide on appropriate follow-up actions.

---

## 5. Enforcement ‚öñÔ∏è

Possible actions in response to violations include, but are not limited to:

- A private warning.
- Asking the person to edit or remove problematic content.
- Temporarily limiting participation in project spaces.
- Blocking / banning the user from the repository in severe or repeated cases.

The maintainer is responsible for applying this Code of Conduct in good faith
and may adapt enforcement actions depending on context and severity.

---

## 6. Acknowledgements üôè

This Code of Conduct is inspired by common open source community guidelines and

If you have suggestions for improvements, please open an issue or pull request
in the repository.
